# ExtendedDeriveDecEq

Computable `DecidableEq` deriver for nested inductive types in Lean 4.

Lean's built-in `deriving DecidableEq` fails on nested inductives, i.e. types that contain
themselves inside a container like `List`, `Option`, or a custom type; yet this is a
common construct in PL mechanisation. This package provides a `derive_deceq` command
that handles them.

## Usage

Add to the `lakefile.toml`:

```toml
[[require]]
name = "extended-derive-deceq"
git = "https://github.com/raoxiaojia/extended-derive-deceq"
rev = "main"
```

Then in a Lean file:

```lean
import ExtendedDeriveDecEq

variable (t : Type) [DecidableEq t]

mutual
inductive Expr where
  | lit : t → Expr
  | app : Expr → List Expr → Expr
  | lam : List Param → Expr → Expr
  | cond : Expr → Expr → Option Expr → Expr
  | typed : Typed → Expr
inductive Param where
  | mk : String → Ty → Option Expr → Param
inductive Ty where
  | base : String → Ty
  | arrow : List Ty → Ty → Ty
inductive Typed where
  | ann : Expr → Ty → Typed
end

derive_deceq Expr   -- generates DecidableEq for all 4 types and auxiliary containers
```

## Scope

`derive_deceq` handles the same input space as Lean's built-in `deriving
DecidableEq` — simple, mutual, indexed, parametric inductives, with
Prop-typed fields — and additionally handles **nested inductives**, i.e.
types that appear recursively inside a container like `List`, `Option`,
`Array`, or a user-defined container. This is the case the built-in
deriver rejects.

## How it works

Unlike Lean's built-in deriver, which analyzes the inductive type definition directly,
this deriver analyzes the recursor generated by the kernel. The recursor encodes the
full mutual block structure, including auxiliary types the kernel creates for nested
containers (e.g. `List Expr` generates auxiliary motives and minors for `List`). This lets
the deriver automatically discover and generate comparison functions for these auxiliary types.

Uses an O(|constructors|) construction based on `ctorIdx` + `casesOnSameCtor`. For each
type, generates one lambda per constructor for same-constructor comparison, then dispatches
cross-constructor disequality in O(1) via `congrArg ctorIdx`.

A O(|motives|^3) Floyd-Warshall pass determines which types are in a recursive call cycle,
so that `termination_by structural` is only emitted where needed, avoiding redundant
termination annotations.

All generated definitions are wrapped in a `mutual ... end` block and elaborated normally,
producing kernel-verified proofs without unsafe bridges.
